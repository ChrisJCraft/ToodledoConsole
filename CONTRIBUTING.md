# Contributing to Toodledo Console

Thank you for your interest in contributing to Toodledo Console! We welcome contributions from the community to help make this tool even better.

## How to Contribute

### Reporting Bugs
If you find a bug, please create an issue on GitHub with a detailed description of the problem, including:
- Steps to reproduce the bug
- Expected behavior
- Actual behavior
- Any relevant error messages or screenshots

### Suggesting Features
Have an idea for a new feature? Please open an issue to discuss it. We value community feedback and are always looking for ways to improve the tool.

### Pull Requests
We welcome pull requests! To contribute code:
1. Fork the repository.
2. Create a new branch for your changes (`git checkout -b feature/your-feature-name` or `bugfix/your-bug-fix`).
3. Make your changes and ensure the code builds (100% clean build with zero warnings is the goal).
4. Commit your changes with a descriptive commit message.
5. Push your branch to your fork.
6. Open a pull request against the `main` branch.

## Coding Standards
- **Spectre.Console**: Follow the existing patterns for UI rendering using Spectre.Console.
- **Service Architecture**: Maintain the modular, service-based architecture.
- **Nullable Reference Types**: Ensure nullable reference types are correctly handled.
- **Documentation**: Update the README if your changes add new commands or features.
